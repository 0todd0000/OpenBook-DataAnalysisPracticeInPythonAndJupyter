Python:   Visualizing 2D arrays


2D arrays usually contain hundreds or thousands of values, and our brains our incapable of understanding the 2D patterns in so many numbers. Matplotlib offers many functions for plotting 2D data, and two key functions are plt.pcolor and plt.imshow.

plt.pcolor¶

pcolor means "psuedo-color", which in turn means that numbers are assigned colors. Here is a simple example:

In [1]:
import numpy as np
from matplotlib import pyplot as plt

a = np.array([  [1,2,3], [4,5,6] ])

plt.figure()
plt.pcolor(a)
plt.colorbar()
plt.show()

Note that the pcolor command produces a pseudo-color plot and that colorbar generates the color bar on the right. Colors represent values.

Note that the first row (with values: [1,2,3]) is on the bottom of the pcolor plot.

We can change various aspects of the pcolor plot including the colormap:

In [2]:
plt.figure()
plt.pcolor(a, cmap='jet')
plt.colorbar()
plt.show()

The value of pcolor becomes more obvious when dealing with larger arrays, and arrays that have spatial structure, like the example below. Note that this example is a modified version of this example from Matplotlib's documentation.

In [3]:
n   = 100
x,y = np.meshgrid(np.linspace(-3, 3, n), np.linspace(-2, 2, n))
z   = np.exp(-x**2 - y**2)

plt.figure()
plt.pcolor(x, y, z, cmap='PuBu_r')
plt.show()

Note that the x values increase from left to right, and that the y values increase from top to bottom.

The $z$ variable is the variable that is actually plotted using pcolor, and it is a function of $x$ and $y$:

$z(x,y) = e^{(-x^2 - y^2)}$

Refer to Matplotlib's pcolor documentation for more pcolor details and options.

plt.imshow¶

An alternative to pcolor is imshow or "image show". Let's see how it works for the examples above:

In [6]:
a = np.array([  [1,2,3], [4,5,6] ])

plt.figure()
plt.imshow(a, cmap='jet')
plt.colorbar()
plt.show()

Note that this is nearly identical to pcolor, except that the first row is on the top. This is the normal convention for images: y values increase from top to bottom.

In [7]:
n   = 100
x,y = np.meshgrid(np.linspace(-3, 3, n), np.linspace(-2, 2, n))
z   = np.exp(-x**2 - y**2)

plt.figure()
plt.imshow(z, cmap='PuBu_r')
plt.show()

This, again, is almost the same as the pcolor result. A key difference is that, whereas pcolor can accept three input array arguments (x, y, z), imshow can only accept one input array argument. Thus imshow, like most image display software, assumes that the values are equally spaced in the $x$ and $y$ directions. pcolor more flexibly allows arbitrary $x$ and $y$ spacing.

A key advantage of imshow is that it can be used to display images, like this:

In [8]:
fname = 'img/1206_FMRI.jpg'
I     = plt.imread(fname)

plt.figure()
plt.imshow(I)
plt.show()

Note that the image file 1206_FMRI.jpg is saved in a directory called img, and that the img directory is in the same directory as this notebook.
